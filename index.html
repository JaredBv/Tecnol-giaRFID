Proyecto Integrador

Innovar, proteger y transformar el futuro digital.

De un simple escaneo a un sistema completo de gestión en segundos.

Transforma el control de asistencia con tecnología RFID: Rápido, seguro y eficiente.

Haz que cada asistente se sienta valorado con un sistema que no falla.

Introducción:

“Nuestro sistema de registro de asistencia basado en tecnología RFID combina innovación y practicidad. Diseñado para instituciones educativas y empresas, permite registrar entradas y salidas de manera automatizada, garantizando un manejo seguro y eficiente de los datos.”

Protección automática: El software está protegido desde el momento de su creación sin necesidad de registro, aunque registrar los derechos puede ser útil en caso de disputas legales.

¿Cómo Funciona?

1.Lectura RFID: Cada usuario pasa su tarjeta RFID frente al lector, que detecta su identificador \u00fanico (UID).

2. Procesamiento: El UID es enviado al servidor a trav\u00e9s del ESP8266, que lo verifica y clasifica como entrada o salida.

3. Visualización: Los datos se reflejan en una interfaz web en tiempo real, permitiendo su consulta inmediata.

Lenguajes de Programación

1. Java Lenguaje para el Backend

En el proyecto del sistema de registro de asistencia basado en RFID, se utilizan diferentes lenguajes de programación para manejar el hardware, el backend y el frontend.

Uso: Desarrollo del servidor backend con el framework Spring Boot.

Propósito:

Procesar las solicitudes HTTP enviadas por el ESP8266.

Gestionar la lógica de negocio, como registrar entradas y salidas.

Conectar y manipular la base de datos para almacenar registros de asistencia.

Frameworks/Librerías Usadas:

Spring Boot (gestión del backend).

Spring Web (manejo de solicitudes HTTP).

Spring Data JPA (conexión con la base de datos).

3. Compatibilidad de licencias: Especialmente en proyectos colaborativos que combinan diferentes licencias.

2. Lenguajes para el Hardware

C++:

Uso: Programación del microcontrolador ESP8266.

Propósito:

Leer datos del lector RFID.

Procesar el UID de las tarjetas.

Enviar datos al servidor backend mediante HTTP.

Herramienta: Arduino IDE.

3. Lenguajes para el Frontend

HTML:

Uso: Estructuración de la interfaz web.

Propósito: Crear las páginas de usuario y administrador para visualizar y gestionar los registros de asistencia.

CSS:

Uso: Diseño y estilo de la interfaz web.

Propósito: Asegurar que la interfaz sea atractiva y fácil de usar.

JavaScript:

Uso: Añadir funcionalidades dinámicas en la interfaz web.

Propósito: Manejar las actualizaciones en tiempo real mediante AJAX. Mejorar la interacción del usuario con el sistema.

Librerías/Frameworks Usadas: AJAX (para actualizar datos sin recargar la página).

4. Lenguajes para la Base de Datos

SQL:

Uso: Gestión de datos en la base de datos relacional (MySQL).

Propósito:

Crear tablas para almacenar usuarios, UID de tarjetas, registros de asistencia y otros datos relevantes.

Ejecutar consultas para filtrar, buscar y exportar información desde la interfaz web.

Áreas para la Formación del Sistema RFID

Esta integración de electrónica, programación y bases de datos asegura que el sistema sea rápido, seguro y eficiente para la gestión de asistencia.

Nube

La nube y la base de datos garantizan el almacenamiento seguro y la disponibilidad de la información en tiempo real.

Electrónica

La electrónica es la base del funcionamiento físico del sistema, integrando componentes esenciales para la captura y transmisión de datos.

Programación

La programación permite que los componentes físicos y digitales trabajen de manera coordinada.

Modelado 3D

El modelado 3D es una herramienta clave para visualizar el diseño físico del sistema, integrando todos los componentes electrónicos y su disposición en el espacio..

Arduino IDE

Modelado como un microcontrolador pequeño en la parte interna de la carcasa, distribuido con el entorno de desarrollo Arduino IDE.

Beneficios de la Tecnología RFID

01
1. Automatización de Procesos

Registro Instantáneo: Los datos se capturan de manera automática y en tiempo real, eliminando la necesidad de procesos manuales.

Eficiencia Operativa: Acelera actividades como el control de asistencia, reduciendo el tiempo de espera en puntos de acceso.

02
2. Precisión y Reducción de Errores

Menos Errores Humanos: Al ser un sistema automatizado, se eliminan los errores comunes asociados con el registro manual, como olvidos o duplicidades.

Trazabilidad Confiable: Cada tarjeta tiene un identificador único (UID), lo que asegura que los registros sean precisos y difíciles de falsificar.

03
3. Seguridad

Identificación Única: Cada tarjeta tiene un UID único que dificulta su clonación o manipulación.

Control de Acceso: Se puede integrar con protocolos de seguridad para permitir o restringir accesos basados en permisos predefinidos.
